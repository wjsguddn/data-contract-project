# 데이터 표준계약 검증 시스템 발표 대본

## 발표 개요
- 발표 시간: 약 15-20분
- 대상: 사업부장님, 그룹장님
- 목적: AI 기반 계약서 검증 시스템의 기술적 우수성과 비즈니스 가치 전달

---

## 1. 오프닝 (1분)

안녕하십니까. 오늘 저희 팀이 개발한 **데이터 표준계약 검증 시스템**에 대해 말씀드리겠습니다.

현재 기업들이 데이터 거래 계약을 체결할 때, 표준계약서와의 정합성을 확인하는 데 많은 시간과 전문 인력이 필요합니다. 저희는 이 과정을 **AI로 자동화**하여, 계약서 검증 시간을 **수 시간에서 수 분으로 단축**하는 시스템을 구축했습니다.

---

## 2. 문제 정의 및 솔루션 개요 (2분)

### 현재의 문제점
기존에는 법무팀이나 계약 담당자가 사용자 계약서를 표준계약서와 일일이 대조하며 검토해야 했습니다. 이 과정에서:
- **시간 소요**: 계약서 1건당 평균 2-3시간
- **일관성 부족**: 검토자마다 판단 기준이 다름
- **누락 위험**: 중요 조항을 놓칠 가능성

### 저희 솔루션
AI 기반 3단계 검증 시스템으로 이 문제를 해결했습니다:
1. **A1 단계**: 조항 매칭 - 어떤 표준 조항과 대응되는지 자동 식별
2. **A2 단계**: 체크리스트 검증 - 필수 항목 포함 여부 확인
3. **A3 단계**: 내용 분석 - 표준 대비 충실도 평가

---

## 3. 핵심 기술: A1 조항 매칭 (4분)

### 기술적 차별점
A1 단계는 시스템의 **기준점**을 만드는 핵심 단계입니다.


#### 양방향 하이브리드 검색
기존 시스템들은 단방향 검색만 사용하지만, 저희는 **양방향 검색**을 도입했습니다.

**정방향 검색**: 사용자 조항 → 표준 조항 검색
- "이 사용자 조항은 어떤 표준 조항과 유사한가?"

**역방향 검색**: 표준 조항 → 사용자 조항 검색  
- "이 표준 조항은 사용자 계약서 어디에 있는가?"

두 방향의 결과가 **일치하는 경우만** 최종 매칭으로 인정합니다.

**효과**:
- 매칭 정확도 **95% 이상** (단방향 대비 20% 향상)
- 오탐지 **50% 감소**

#### 하이브리드 검색 기술
단순히 키워드만 보거나 의미만 보는 것이 아니라, 두 가지를 결합했습니다:

**Dense 검색 (85%)**: AI 임베딩으로 의미 유사도 측정
- "데이터 제공 범위"와 "정보 제공 대상"이 의미적으로 유사함을 인식

**Sparse 검색 (15%)**: 키워드 매칭으로 정확도 보완
- "제3조", "데이터" 같은 핵심 키워드 일치 확인

이 조합으로 **의미는 같지만 표현이 다른 조항**도 정확히 찾아냅니다.

#### 누락 조항 재검증
단순히 "매칭 안 됨"으로 끝나지 않습니다.

매칭되지 않은 표준 조항에 대해 **LLM이 재검증**합니다:
- "정말 누락된 것인가?"
- "다른 조항에 통합되어 있지 않은가?"
- "다른 표현으로 작성되어 있지 않은가?"

**결과**: 실제 누락 조항만 정확히 식별 (신뢰도 95% 이상)

---

## 4. A2 체크리스트 검증 (3분)

### 활용안내서 기반 검증
정부에서 제공하는 **활용안내서**에는 각 표준계약서 유형별로 필수 체크리스트가 있습니다.

A2 단계는 이 체크리스트를 기반으로 **45개 이상의 필수 항목**을 자동 검증합니다.


### 효율성 극대화: 중복 제거
예를 들어, "계약 당사자가 명확히 기재되어 있는가?"라는 항목이 제1조에서 YES로 확인되면, 이후 조항에서는 **재검증하지 않습니다**.

**효과**:
- 검증 시간 **30% 단축**
- LLM API 비용 **30% 절감**
- 일관성 향상

### 4단계 판정 시스템
단순히 통과/미통과가 아닌, **4가지 결과**로 세분화했습니다:

1. **YES**: 명확히 충족 (근거 제시)
2. **NO**: 명확히 미충족 (누락 상세, 위험도, 개선 제안)
3. **UNCLEAR**: AI 판단 불가 (수동 검토 필요)
4. **MANUAL_CHECK_REQUIRED**: 외부 문서 대조 필요 (예: 등기부등본 확인)

### 신뢰도 기반 재검증
AI가 확신하지 못하는 경우(신뢰도 < 0.7), **표준 조항을 추가 컨텍스트로 제공**하여 재검증합니다.

"표준계약서에는 이렇게 되어 있는데, 사용자 계약서는 어떤가요?"

**효과**: 애매한 판단의 정확도 **20% 향상**

### 상세 피드백
NO 판정 시, 단순히 "미충족"이 아니라:
- **누락 상세**: 구체적으로 무엇이 빠졌는지
- **위험도**: high/medium/low (법적 리스크 평가)
- **개선 제안**: 어떻게 보완해야 하는지

**예시**:
```
누락: 데이터 제공 주기가 명시되지 않음
위험도: high - 실무적 혼란 가능성
제안: 제3조에 "월 1회" 등 구체적 주기 명시
```

## 5. A3 내용 분석 (3분)

### 맥락 기반 유연한 검증
A3 단계의 핵심 철학은 **"표준계약서는 템플릿이지 정답이 아니다"**입니다.


#### 구체화를 긍정적으로 평가
표준계약서: "데이터 항목: [구체적 항목 명시]"
사용자 계약서: "데이터 항목: 고객 성명, 연락처, 구매 이력"

→ 이것은 **문제가 아니라 개선**입니다. 표준의 포괄적 표현을 실제 상황에 맞게 구체화한 것이기 때문입니다.

#### 의미 중심 평가
- 표현 형식 차이 (목록 vs 서술형) → 무시
- 단어/순서/문체 차이 → 무시
- **핵심 의미가 포함되어 있는가?** → 이것만 평가

#### 조 단위 종합 비교
항 단위가 아닌 **조 전체를 종합**하여 평가합니다.

**단일 매칭**: 사용자 제3조 vs 표준 제3조 전체 비교
**다중 매칭**: 사용자 제5조 vs 표준 제5조+제6조 종합 비교

**효과**: 조항 간 관계를 고려한 실질적 평가 가능

### 심각도 자동 판정
누락/불충분 항목 개수를 기반으로 **자동 등급화**:

- **high**: 누락 3개 이상 또는 합계 5개 이상 → 즉시 보완 필요
- **medium**: 누락 2개 이상 또는 불충분 2개 이상 → 보완 권장
- **low**: 그 외 → 선택적 보완
- **info**: 문제 없음 → 긍정적 피드백

### 긍정적 분석 포함
문제가 없는 경우에도 **"잘 작성되었습니다"**라는 피드백을 제공합니다.

**효과**: 사용자에게 확신을 주고, 계약서 품질에 대한 신뢰 향상

## 6. 시스템 아키텍처 및 성능 (3분)

### 마이크로서비스 아키텍처
각 검증 단계를 **독립적인 서비스**로 구성했습니다:


- **FastAPI**: REST API 서버 (포트 8000)
- **Streamlit**: 웹 인터페이스 (포트 8501)
- **Redis**: 메시지 큐 (비동기 작업 관리)
- **Celery Workers**: 각 Agent 독립 실행

**장점**:
- 각 단계 독립적 확장 가능
- 장애 격리 (한 단계 오류가 전체 시스템에 영향 없음)
- 병렬 처리로 성능 최적화

### 병렬 처리 최적화
A1 단계가 완료되면, **A2와 A3가 동시에 실행**됩니다.

```
A1 (조항 매칭) → 결과 저장
                    ↓
        ┌───────────┴───────────┐
        ↓                       ↓
    A2 (체크리스트)         A3 (내용 분석)
        ↓                       ↓
        └───────────┬───────────┘
                    ↓
              결과 통합
```

**효과**: 전체 처리 시간 **30% 단축**

### 성능 지표
실제 15개 조항 계약서 기준:

| 단계 | 처리 시간 | 정확도 |
|------|----------|--------|
| A1 조항 매칭 | 30-45초 | 95% 이상 |
| A2 체크리스트 | 20-30초 | 95% 이상 |
| A3 내용 분석 | 35-50초 | 90% 이상 |
| **전체** | **약 1분 30초** | **93% 이상** |

기존 수작업 대비 **95% 이상 시간 단축**

### 비용 효율성
Azure OpenAI API 사용 최적화:
- 배치 처리로 API 호출 최소화
- 중복 검증 제거로 비용 30% 절감
- 캐싱으로 반복 쿼리 제거

**예상 비용**: 계약서 1건당 약 **500-800원** (GPT-4 기준)

## 7. 기술 스택 및 확장성 (2분)

### 핵심 기술
- **AI/ML**: Azure OpenAI (GPT-4, text-embedding-3-large)
- **검색**: FAISS (벡터 검색) + Whoosh (키워드 검색)
- **Backend**: FastAPI, Celery, Redis, SQLAlchemy
- **Frontend**: Streamlit
- **Infrastructure**: Docker, Docker Compose


### 확장 가능성
현재 시스템은 **5종 데이터 표준계약서**를 지원하지만, 쉽게 확장 가능합니다:

1. **다른 계약 유형 추가**: 표준계약서만 추가하면 자동 지원
2. **다국어 지원**: 영어 계약서 처리 가능 (임베딩 모델 다국어 지원)
3. **커스텀 체크리스트**: 기업별 자체 검증 기준 추가 가능
4. **VLM 통합**: 비정형 계약서 구조 인식 (향후 계획)

### 데이터 보안
- **온프레미스 배포 가능**: 민감한 계약서 데이터를 외부로 전송하지 않음
- **Azure 한국 리전 사용**: 데이터 주권 준수
- **암호화**: 저장 데이터 및 전송 데이터 암호화

## 10. 클로징 (1분)

저희가 개발한 **데이터 표준계약 검증 시스템**은 단순한 자동화 도구가 아닙니다.

**3단계 정밀 검증 (A1-A2-A3)**을 통해:
- A1: 양방향 하이브리드 검색으로 **95% 이상 매칭 정확도**
- A2: 중복 제거 및 신뢰도 기반 재검증으로 **효율성 30% 향상**
- A3: 맥락 기반 유연한 검증으로 **실질적 내용 평가**


**비즈니스 임팩트**:
- 검증 시간 **98% 단축** (2-3시간 → 1.5분)
- 연간 비용 **약 2,000만원 절감** (100건 기준)
- 계약 리스크 최소화 및 데이터 거래 활성화

**기술적 차별성**:
- 국내 최초 양방향 하이브리드 검색 적용
- 맥락 기반 유연한 검증 (과도한 규격화 방지)

## 예상 질문 및 답변

### Q1: 기존 법률 검토 서비스와의 차이는?
**A**: 기존 서비스는 일반적인 법률 검토에 초점을 맞추지만, 저희는 **표준계약서 정합성 검증**에 특화되어 있습니다. 정부 표준계약서와 활용안내서를 지식베이스로 활용하여, 데이터 거래 계약에 최적화된 검증을 제공합니다.

### Q2: AI 판단의 신뢰성은?
**A**: 3단계 검증으로 신뢰성을 확보했습니다:
1. **양방향 검색**: 정방향+역방향 일치 확인으로 오탐지 50% 감소
2. **신뢰도 기반 재검증**: 확신 없는 경우 추가 컨텍스트로 재판단
3. **수동 검토 플래그**: AI가 판단할 수 없는 항목은 명시적으로 표시

전체 정확도 **93% 이상**이며, 불확실한 경우 사용자에게 명확히 알립니다.

### Q3: 다른 계약 유형으로 확장 가능한가?
**A**: 네, 매우 쉽게 확장 가능합니다. 표준계약서와 활용안내서만 추가하면, 동일한 파이프라인으로 자동 지원됩니다. 현재 5종 데이터 계약을 지원하지만, 근로계약, 임대차계약 등 다른 유형도 동일한 방식으로 추가할 수 있습니다.

### Q4: 비정형 계약서도 처리 가능한가?
**A**: 현재는 구조화된 DOCX 파일("제n조" 형식)만 지원하지만, Phase 2에서 **VLM(Vision Language Model)** 기반 파싱을 도입하여 비정형 계약서도 처리할 계획입니다. 표, 이미지, 자유 형식의 계약서도 자동 인식할 수 있습니다.


### Q5: 개인정보 보호는 어떻게 하나?
**A**: 
- **온프레미스 배포**: 민감한 계약서를 외부로 전송하지 않고 자체 서버에서 처리
- **Azure 한국 리전**: 데이터 주권 준수
- **암호화**: 저장 및 전송 데이터 모두 암호화
- **접근 제어**: 역할 기반 권한 관리

### Q6: 도입 비용은?
**A**: 
- **초기 구축**: 약 3,000만원 (시스템 구축 + 커스터마이징)
- **운영 비용**: 월 약 50만원 (Azure OpenAI API + 서버 운영)
- **ROI**: 연간 100건 처리 시 약 1,992만원 절감 → **1년 내 투자 회수**

### Q7: 법무팀의 역할은?
**A**: AI가 **1차 검증**을 수행하고, 법무팀은 **전략적 검토**에 집중합니다:
- AI 검증 결과 최종 승인
- 고위험 항목 집중 검토
- 협상 전략 수립
- 예외 사항 판단

단순 반복 작업에서 해방되어 **고부가가치 업무**에 집중할 수 있습니다.

### Q8: 경쟁사 대비 우위는?
**A**: 
1. **양방향 하이브리드 검색**: 국내 최초 적용, 정확도 20% 향상
2. **맥락 기반 검증**: 과도한 규격화 방지, 실질적 평가
3. **3단계 정밀 검증**: 완전성(A1) + 체크리스트(A2) + 내용(A3)
4. **정부 표준 기반**: 공신력 있는 표준계약서 활용
5. **확장성**: 마이크로서비스 아키텍처로 유연한 확장
